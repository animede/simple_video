# かんたん動画 チュートリアル

このチュートリアルは、各機能を最短で試す手順です。

## 1. 起動

```bash
cd simple_video_app
./start.sh
```

ブラウザ: `http://127.0.0.1:8090/`

## 2. 動画を作る（T2V）

1. `⚙️ 生成シーケンス` で T2V 系を選択
2. `⚙️ 動画設定` でシーン数・長さを調整
3. `📜 シナリオ入力` に作りたい内容を記述
4. 必要なら `🎨 画風テンプレ` を押してスタイルを注入（適用時はシーン長の標準値が `5秒` になります）
5. `🧠 シナリオ作成` → `🤖 プロンプト生成` の2ステップでシーンプロンプトを作成
6. `動画を生成` を押す
7. `🎬 出力動画` に結果が表示される

## 3. キャラクタ動画を作る（I2V / FLF）

1. `📥 画像ドロップ` に参照画像を入れる（ref1〜ref3）
2. `👤 キャラクタ一覧` で選択、または `📝 キャラクタ登録` で新規登録
3. `📝 何を描きたい？` に初期画像プロンプトを入力
4. `初期画像を生成` を押す（生成後にキー画像として自動反映）
5. `⚙️ 生成シーケンス` でキャラクタ系（I2V / FLF）を選択
6. `動画を生成` を押す

補足:

- `char_edit_i2i_flf` では初期画像・キャラ画像が特に重要です
- 中間画像確認が有効な場合、`CONTINUE` で後段へ進みます

## 4. 混在トランジション動画を作る（I2I + FLF/I2V 自動選択）

「キー画像参照 I2I + 混在トランジション」または「キャラ EDIT + I2I + 混在トランジション」を使うと、
LLMがシーン間の関係を判定して FLF（滑らかな遷移）と I2V（動き重視）を自動で使い分けます。

### キー画像参照版（キャラ不要）

1. `🖼️ キー画像` に参照画像をアップロード
2. `⚙️ 生成シーケンス` で `🔀 キー画像参照 I2I + 混在トランジション` を選択
3. `⚙️ 動画設定` でシーン数を設定（内部で **N+1枚** の画像が生成されます）
4. シナリオを入力し、`🤖 プロンプト生成` を実行
   - 生成後、プロンプト欄の下に `🔀 シーン遷移タイプ` エディタが展開します
   - 各シーンの遷移タイプ（FLF / カット / クロスフェード / フェード黒）を必要に応じて変更可能
5. `🖼️ シーン事前生成` を押してN+1枚のシーン画像を生成
   - 中間画像は確認・差し替え可能です
6. `CONTINUE` で動画生成フェーズへ進む（FLF または I2V が境界ごとに実行）
7. 結合後の動画が `🎬 出力動画` に表示される

### キャラ EDIT 版

上記と同じ手順ですが、`🖼️ キー画像`（キャラ画像）とキャラ選択/登録が必要です。  
EDIT合成→I2Iシーン画像という2段階でキャラ一貫性がより高くなります。

補足:

- `🔀 シーン遷移タイプ` エディタはアコーディオン形式（折りたたみ）で表示。手動変更は即時保存されます
- LTX系ワークフローは現状このプリセットでは利用できません（I2V部分は WAN 固定）

## 5. 音楽を作る（T2A）

1. 音楽生成の入力欄に曲の雰囲気や歌詞イメージを入力
2. 必要に応じて歌詞・タグ提案を使う
3. 音楽生成を実行して、生成音源を確認

## 6. 音楽から動画を作る（M2V）

1. 生成音源またはアップロード音源を選択
2. 動画シナリオを入力
   - 未入力時は確認ダイアログが表示されます
   - `このまま生成する` または `シナリオを入力する` を選択
3. `動画を生成` を実行
4. 最終成果物は `output/movie` に保存されます

## 7. 動画から音楽を作る（V2M）

1. 生成済み動画またはアップロード動画を選択
2. 必要に応じて音楽スタイル条件を指定
3. 実行して、動画に合成された音楽結果を確認
4. 最終成果物は `output/movie` に保存されます

## 8. つまずいたとき

- `ComfyUI /prompt failed` が出たら、エラー詳細の「どの設定や入力で止まったか」を確認（`node_errors` 欄に原因が出ます）
- 画像参照エラー時は、`📥 画像ドロップ` とキャラ選択状態を再確認
- 古い job_id の 404 は、再生成すると解消されることがあります
