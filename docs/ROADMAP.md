# Simple Video Roadmap（開発ロードマップ）

このドキュメントは、今後の改良候補を記録する内部メモです。  
実装時期は未定です。優先度と実現性を見ながら段階的に進めます。

---

## 1. シーン切り替えトランジション混在

**目的**: ブツ切り連結と FLF 一辺倒を脱却し、プロの映像に近いテンポ感を実現する

**現状の課題**:

| 方式 | 特徴 | 問題 |
|------|------|------|
| FLF 連続 | 滑らかだが全シーンが地続き | 場面転換の「切れ」がない。テンポが単調 |
| シーンカット | ブツ切りで連結 | 唐突。視聴者に「区切り」の意図が伝わらない |

**段階的アプローチ**:

### Phase 1: ffmpeg xfade（低コスト・即実装可能）

- シーン間に ffmpeg トランジションを挿入: `xfade`（crossfade / dissolve / wipe / fade 等）
- LLM にシーンごとの推奨トランジションタイプを出力させる
- 実装コスト: 低。`ffmpeg -filter_complex xfade` で連結時に適用するだけ
- **生成モデル側の変更不要**

### Phase 2: LLM 判定 + FLF/カット混在パイプライン

- LLM がシーンの関係性を判定し、トランジション種別を自動選択:
  - `"transition": "flf"` → FLF で滑らかに繋ぐ
  - `"transition": "cut"` → ハードカット
  - `"transition": "crossfade"` → ffmpeg xfade
  - `"transition": "fade_black"` → 暗転
- 1本の動画内で FLF 区間とカット区間が混在するパイプライン

### Phase 3: 可変シーン長との組み合わせ

- Phase 2 + 可変シーン長を統合
- サビは長め＋FLF連続、Aメロは短め＋カット切り替え、のような構成

**優先度**: 高（視聴体験への効果が大きく、Phase 1 は低コストで実装可能）

## 2. 可変シーン長（シーンごとの尺調整）

**目的**: 物語の進行に合わせて、大事なシーンを長く、クローズアップを短くする

**現状の課題**:

- 全シーン固定長（2s / 3s / 4s / 5s / 8s から選択）
- 感情のピークが間延びし、テンポの良いカットが駆け足になる
- MV では楽曲フレーズ構造との同期が取れない

**段階的アプローチ**:

### Phase 1: LLM に推奨秒数を出力させる（UI 表示のみ）

- プロンプト生成の出力フォーマットに `duration` フィールドを追加
- 例: `{"scene": 1, "duration": 5, "reason": "establishing shot"}`
- ユーザーが UI 上で確認・手動調整
- 生成ワークフロー側の変更は最小限

### Phase 2: 推奨秒数の自動適用

- LLM の推奨値を自動適用
- Wan2.2 のフレーム数制約（81/121 frames 等）に丸める（2s/3s/4s/5s/8s の最近値）
- シーンごとに異なる `num_frames` をワークフローに渡す

### Phase 3: M2V での楽曲同期

- 楽曲構造（BPM・小節）を解析し、フレーズ境界に合わせてシーン長を自動配分
- イントロ / Aメロ / サビ / ブリッジ / アウトロの区間に映像シーンをマッピング

**技術的な難しさ**:

| 課題 | 理由 |
|------|------|
| LLM による尺判定 | 「このシーンは何秒が最適か」の正確な判定は非自明 |
| Wan2.2 の制約 | 生成可能なフレーム数が離散的で、任意秒数に対応しにくい |
| FLF 遷移の整合性 | シーン長が異なると遷移タイミングの設計が複雑化 |
| M2V の楽曲同期 | BPM・小節単位での分割が必要 |

**優先度**: 中（効果は大きいが、技術的ハードルがトランジションより高い）

## 優先順位まとめ

| 順位 | 機能 | Phase | 理由 |
|------|------|-------|------|
| 1 | トランジション混在 | Phase 1 | ffmpeg だけで即実装可能。視聴体験への効果大 |
| 2 | トランジション混在 | Phase 2 | LLM 判定の追加。FLF/カット混在 |
| 3 | 可変シーン長 | Phase 1 | LLM 出力フォーマット追加のみ。低リスク |
| 4 | 可変シーン長 | Phase 2 | ワークフロー側の変更が必要 |
| 5 | 統合 | Phase 3 | 両機能の統合 + M2V 楽曲同期 |
